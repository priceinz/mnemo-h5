## 三、实施方案A：纯对话式

> 适合：**想立刻用起来，等不及配置工作流**

### 3.1 创建AI助理

**路径**：钉钉 → AI助理 → 创建助理

| 配置项 | 填写内容 |
|--------|----------|
| 助理名称 | **审计团队管家** |
| 助理描述 | 管理7人审计团队的签到、日报情况 |
| 图标 | 选管理/报表类图标 |
| 模型 | 推荐「通义千问」或「DeepSeek-V3」 |

### 3.2 配置角色设定

在「角色设定」框中粘贴以下内容（**记得把姓名换成真实的**）：

```
你是「审计团队管家」，是一位专业的团队管理助理。你负责管理一个7人审计团队，帮助领导掌握团队每日工作动态。

【团队成员】
- 工程审计组（1人）：张三
- 供应监察组（2人）：李四、王五
- 内部审计组（4人）：赵六、钱七、孙八、周九

【你的职责】
1. 签到监控：跟踪团队成员每日外勤签到情况，识别未签到人员
2. 日报管理：跟踪日报提交情况，总结日报内容要点
3. 异常预警：对未签到、未交日报的人员进行提醒
4. 定期汇总：生成每日/每周/每月的团队工作汇总

【汇报格式规范】

当汇报每日情况时，使用以下格式：

📋 每日汇总 — [日期]

📁 工程审计
[姓名]
- 签到：✅已签到 [地点][时间] / ⚠️未签到
- 日报：✅已提交 / ❌未提交
- 摘要：[日报内容的一句话总结]

📁 供应监察
（同上格式）

📁 内部审计
（同上格式）

📊 统计：签到 X/7 | 日报 X/7

当汇报异常时，使用以下格式：

🚨 异常预警 — [日期]
⚠️ 未签到（X人）：[按组列出姓名]
❌ 未交日报（X人）：[按组列出姓名]

当汇报周报时，使用表格格式，包含：姓名、组别、本周签到次数、日报提交数/5、完成率

当汇报月报时，使用表格格式，包含：姓名、组别、签到次数、日报提交数/工作日数、日报率，并用 🟢(≥90%) 🟡(70-89%) 🔴(<70%) 标识

【交互规则】
- 当领导问"今天情况怎么样"，输出每日汇总
- 当领导问"谁还没签到/交日报"，输出异常预警
- 当领导问"这周/本周汇总"，输出周报
- 当领导问"上个月汇总"，输出月报
- 当领导说"提醒XX交日报"，生成一条可转发的提醒消息
- 保持简洁专业，不要过度寒暄
- 如果领导提供了不完整的数据，主动追问缺失的人员信息
- 记住历史对话中的数据，用于生成周报和月报汇总
```

### 3.3 添加快捷指令

在「快捷指令」中添加：

| 指令名称 | 触发词 | 功能 |
|---------|--------|------|
| 今日汇总 | /今日 | 生成今日签到+日报汇总 |
| 异常预警 | /异常 | 列出未签到和未交日报人员 |
| 周报 | /周报 | 生成本周汇总表格 |
| 月报 | /月报 | 生成本月汇总表格 |
| 提醒交日报 | /催报 [姓名] | 生成催交日报的消息 |

## 四、实施方案B：工作流自动化

> 满足你的需求：**9:00自动提醒缺卡 / 21:00自动提醒日报**

### 4.1 创建AI助理

与方案A相同，创建「审计团队管家」助理。

### 4.2 工作流一：9:00 缺卡提醒 ⏰

**目标**：每天9:00自动检测谁没打卡，给没打卡的人发提醒

```
工作流名称：早9点缺卡提醒
触发方式：定时触发
触发时间：每天 09:00
```

**流程编排：**
```
[定时触发: 每天09:00]
    ↓
[HTTP请求: 获取今日签到记录]
    ↓
[代码节点: 对比员工列表，找出未签到人员]
    ↓
[判断节点: 有未签到人员？]
    ├─ 是 → [循环节点: 给每个未签到人员发消息]
    └─ 否 → [结束]
```

**HTTP请求节点配置：**
```
请求方式：POST
请求地址：https://oapi.dingtalk.com/topapi/checkin/record/get
参数 access_token：${从连接器获取}

Body(JSON)：
{
  "userid_list": "${员工user_id列表}",
  "start_time": "${今天0点的毫秒时间戳}",
  "end_time": "${当前毫秒时间戳}",
  "cursor": 0,
  "size": 100
}
```

**代码节点（找出未签到人员）：**
```javascript
// 员工配置（替换为真实的user_id和姓名）
const employees = [
  {name: "张三", userid: "实际user_id", group: "工程审计"},
  {name: "李四", userid: "实际user_id", group: "供应监察"},
  {name: "王五", userid: "实际user_id", group: "供应监察"},
  {name: "赵六", userid: "实际user_id", group: "内部审计"},
  {name: "钱七", userid: "实际user_id", group: "内部审计"},
  {name: "孙八", userid: "实际user_id", group: "内部审计"},
  {name: "周九", userid: "实际user_id", group: "内部审计"},
];

// 获取签到记录
const checkinList = input.result?.page_list || [];
const checkedUserIds = new Set(checkinList.map(r => r.userid));

// 找出未签到人员
const absentEmployees = employees.filter(e => !checkedUserIds.has(e.userid));

// 生成提醒消息
const reminders = absentEmployees.map(emp => ({
  userid: emp.userid,
  name: emp.name,
  message: `⏰ 打卡提醒\n\n${emp.name}，早上好！\n\n检测到您今日尚未打卡，请及时补卡。\n如有外勤安排，请备注说明。`
}));

return {
  hasAbsent: absentEmployees.length > 0,
  absentCount: absentEmployees.length,
  reminders: reminders
};
```

**发消息节点：**
- 接收人：`${reminders.userid}`（循环变量）
- 消息内容：`${reminders.message}`（循环变量）

### 4.3 工作流二：21:00 日报汇总+提醒 ⏰

**目标**：每天21:00自动检查日报，给管理者发汇总，给未交者发提醒

```
工作流名称：晚9点日报监控
触发方式：定时触发
触发时间：每天 21:00
```

**流程编排：**
```
[定时触发: 每天21:00]
    ↓
[HTTP请求: 获取今日日报列表]
    ↓
[代码节点: 整理日报数据，找出未提交人员]
    ↓
[发消息节点: 给管理者推送日报汇总]
    ↓
[循环节点: 给每个未提交人员发提醒]
```

**HTTP请求节点配置：**
```
请求方式：POST
请求地址：https://oapi.dingtalk.com/topapi/report/list
参数 access_token：${从连接器获取}

Body(JSON)：
{
  "start_time": "${今天0点的毫秒时间戳}",
  "end_time": "${当前毫秒时间戳}",
  "template_name": "日报",
  "cursor": 0,
  "size": 100
}
```

**代码节点（整理日报数据）：**
```javascript
// 员工配置（替换为真实的user_id和姓名）
const employees = [
  {name: "张三", userid: "实际user_id", group: "工程审计"},
  {name: "李四", userid: "实际user_id", group: "供应监察"},
  {name: "王五", userid: "实际user_id", group: "供应监察"},
  {name: "赵六", userid: "实际user_id", group: "内部审计"},
  {name: "钱七", userid: "实际user_id", group: "内部审计"},
  {name: "孙八", userid: "实际user_id", group: "内部审计"},
  {name: "周九", userid: "实际user_id", group: "内部审计"},
];

// 获取日报记录
const reports = input.result?.data_list || [];
const reportMap = {};

reports.forEach(r => {
  reportMap[r.creator_id] = r;
});

// 按组整理
const groups = {};
let submittedCount = 0;

employees.forEach(emp => {
  if (!groups[emp.group]) groups[emp.group] = [];

  const report = reportMap[emp.userid];
  const submitted = !!report;
  if (submitted) submittedCount++;

  // 提取摘要
  let summary = "";
  if (report && report.contents && report.contents.length > 0) {
    const val = report.contents[0].value || "";
    summary = val.length > 60 ? val.substring(0, 60) + "..." : val;
  }

  groups[emp.group].push({
    name: emp.name,
    submitted,
    summary
  });
});

// 生成汇总文本
const today = new Date().toLocaleDateString("zh-CN", {month: "long", day: "numeric"});
let summaryText = `📋 每日日报汇总 — ${today}\n\n`;

for (const [group, members] of Object.entries(groups)) {
  summaryText += `📁 ${group}\n`;
  members.forEach(m => {
    if (m.submitted) {
      summaryText += `  ${m.name}：✅ 已提交\n`;
      if (m.summary) summaryText += `    └ ${m.summary}\n`;
    } else {
      summaryText += `  ${m.name}：❌ 未提交\n`;
    }
  });
  summaryText += "\n";
}

summaryText += `📊 统计：日报 ${submittedCount}/7`;

// 找出未提交人员
const unsubmitted = employees.filter(e => !reportMap[e.userid]);

return {
  summaryText,
  submittedCount,
  totalCount: employees.length,
  unsubmitted,
  hasUnsubmitted: unsubmitted.length > 0
};
```

**发消息节点1（给管理者）：**
- 接收人：你的user_id
- 消息内容：`${summaryText}`

**发消息节点2（给未提交者，循环）：**
- 接收人：`${unsubmitted.userid}`
- 消息内容：
```
📝 日报提醒

${unsubmitted.name}，您好！

检测到您今日日报尚未提交，请在24:00前完成提交。

谢谢配合！
```

### 4.4 关于 access_token 获取

**方式一：钉钉连接器（推荐，免代码）**

1. 在工作流中添加「连接器」节点
2. 搜索「钉钉官方连接器」
3. 选择「签到」或「日志」相关连接器
4. 按提示授权即可

**方式二：手动创建应用**

如果要用HTTP节点直接调用：

1. 打开 https://open.dingtalk.com → 扫码登录
2. 「应用开发」→「企业内部应用」→「创建应用」
3. 名称填「团队管理助理」
4. 进入应用 →「基础信息」→ 复制 AppKey 和 AppSecret
5. 「权限管理」→ 开通：签到数据读权限、日志相关权限
6. 「版本管理与发布」→ 发布
